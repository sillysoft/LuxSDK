LuxSDK AI Info

This SDK contains everything you need to write your own computer AI for Lux. The SDK uses the Java programming language and is made up of a series of public API as well as examples (the source for all the AIs that come with Lux are included).

The LuxAgent Interface
Lux is written in Java, and each agent is just a class that implements the LuxAgent interface. The interface comes with lots of comments, and should probably be the first thing you examine.

If you are unfamiliar with java interfaces, it is basically just a list of function prototypes. To write a class that implements the interface, you just fill in the code for all of the prototypes.

The Lux API
Other then the LuxAgent interface, agents must interact with Lux. This is done using the public Lux API. Have a look at the Lux API javadocs to see what is available.

Examples
The source code of many Lux agents are included. Details can be found on the example agents page. The code is licensed to you either under the GNU GPL or free for non profit use (your choice).

To Compile
Method #1: Use the included apache ant build.xml file to build using ant.
Method #2: Copy the java files inside src into your preferred IDE.

Finishing Up
Once you have finished writing an agent, place your compiled .class file into ~/Library/Application Support/Lux/Agents/ (for Mac OS X users) or into the Support/Agents/ folder where you installed Lux (i.e. C:/Program Files/Lux/Support/Agents/). Your agent should now appear in the agent select-boxes when Lux is run. If you use ant to compile then modify the build.xml file to specify your support folder and you can automatically move the newly compiled file into place.

 

Random Notes:
Often I will talk of a 'cluster', usually specified by a root country. A cluster is a group of countries that all have the same owner and can all be reached from each other.

When Lux is running, all method calls into agents are inside try/catch statements. This means that errors within agents will not cause Lux to crash. Whenever an exception is caught it is printed to standard out including a stacktrace.

As well, a message is printed to standard out whenever an agent sends an improper command. (eg. it tries to attack a country that it owns)

(Under MacOS X, standard out of all the running applications can be seen with the Console application, located in /Applications/Utilities/

For testing purposes it is sometimes useful to run game after game after game... Start Lux from the command line and pass it the -superfast option to get a mode that uses less graphics. You'll still have to start a game and turn on the Autoplay feature from the Game menu. The Autoplay setting doesn't save itself, so it must be set each time you run Lux.

You can find other BotMasters and discussion in Lux's Robot House Forum.
